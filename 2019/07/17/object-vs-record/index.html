对象与记录

Wed Jul 17 2019

从 c 到 c++ 面向对象：

  1. 所谓“类”和“对象”，基本是过程式语言里面的记录（record，或者叫结构，structure），它本质其实是一个从名字到数据的“映射表”（map）。 例如我们可以在c中写出这样的代码:
struct persion {
	char *name;
	int age;
	void printme(void) { // 在结构体中直接定义了一个函数，使用 g++ 可以编译通过
		printf("name: %s, age: %d\n", name, age);
	}
};
  1. 最大的特点在于提供数据访问时的一层抽象，可以重新定义数据的实现方式，而提供固定的表象给调用者；然而这也导致了数据的不确定性，我们不能再调用的时候真正知道数据的实际意义
  2. 过多的抽象。很多面向对象语言号称“所有东西都是对象”（Everything is an Object），把所有函数都放进所谓对象里面，叫做“方法”（method），把普通的函数叫做“静态方法”（static method）。而实际上，大部分的函数都不属于对象。
  3. 常常有新的理论为了修复过多抽象的弊病而提出来。
  4. 好的语言提供最直接简洁的对现实建模的能力，而不是提供复杂的理论
