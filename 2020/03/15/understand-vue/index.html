# 如何追踪变化

Vue 依靠Object.definedProperty劫持对象属性变化，Object.defineProperty 是 ES5 中一个无法 shim 的特性，这也就是 Vue 不支持 IE8 以及更低版本浏览器的原因；

检测变化的注意事项 受现代 JavaScript 的限制 (而且 Object.observe 也已经被废弃)，Vue 无法检测到对象属性的添加或删除。

Vue会在初始化实例的时候将仅 data 对象上面的属性转为响应式的；

特殊的需要动态添加响应式的情况，参考列表渲染

异步更新队列 监听到数据变化之后，watcher并不会马上通知渲染，而是把请求放入到队列中，

这样做可以防止组件中的一个函数多次频繁渲染（比如for循环中更新响应式数据）；

这是一种数据驱动的编码范式，通常情况下是对性能是最友好的；

在数据与视图渲染频繁交错的情况，比如说下一步的数据处理等待着渲染的完成，这时候可以使用

Vue.$nextTick 
vm.message = 'new message' // 更改数据
vm.$el.textContent === 'new message' // false
Vue.nextTick(function () {
  vm.$el.textContent === 'new message' // true
})

# key的作用

key的作用主要是为了高效的更新虚拟DOM；

createKeyToOldIdx 复制制作 key-index 的字典；

function createKeyToOldIdx(children, beginIdx, endIdx) {
  let i, key
  const map = {}
  for (i = beginIdx; i <= endIdx; ++i) {
    key = children[i].key
    if (isDef(key)) map[key] = i
  }
  return map
}
