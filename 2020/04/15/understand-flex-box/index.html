理解flex布局

Wed Apr 15 2020

flex 布局的初衷是提供容器（父布局）调整子元素尺寸的能力，已达到自动填充和适配子元素之间的间隙的目的;

# 理解flex布局

# 方向轴

flex 布局先要确认轴向，分别有主轴（main-axis） 和 垂直于主轴的纵轴（cross-axis） 为了方便记忆，只需关注主轴； 轴的方向由 flex-direction 决定，可以是 水平的（row），也可以是竖直的（column）

# 理解对齐

align-items、和 justify-content 两个属性决定单行情况下对齐方式； align-items 描述主轴的分布行为，就是一般排版中的概念（居中、靠左、靠右等）； justify 描述的是分布的概念，即控制的是容器里面项目的间隙，需要配合实际的例子体会 between、around 等词汇

align-content、justify-items 决定多行情况下，行与行之间的对齐方式； 其实就是直接看作是纵轴的分布行为

flex布局需要容器和子元素一起设置才能发挥作用，因此两边各自有控制不同表现的属性

如果某一个其中的子布局要求脱离对齐规则，使用 align-self，改变父布局的align-items行为，使用子元素自己的布局行为

align-self: auto | flex-start | flex-end | center | baseline | stretch;

# 容器属性

flex-direction，用作控制方向; flex-direction: row | row-reverse | column | column-reverse;

flex-wrap，子元素默认会挤到一行显示，flex-wrap 改变这一行为，具有自动分行作用;

flex-wrap: nowrap | wrap | wrap-reverse;

flex-flow = flex-direction + flex-wrap flex-flow: <‘flex-direction’> || <‘flex-wrap’>

# 子布局属性

flex-grow，在有剩余空隙的情况下，可以扩展的权重比例

flex-grow: <'number'>; /* default 0 */

flex-strink，与 flex-grow 类似，在空间不够的时候的压缩比;

flex-wrap, 决定分行以及分行后显示的方向；

flex-wrap: nowrap | wrap | wrap-reverse;

# 简写，

flex: 1 是 flex-grow: 1, flex-shrink, flex-basis 的简写； flex: 1 表示 flex-grow: 1;

# 预设

# 横向布局

<view class="flex">
    <view>xxx</view>
    <view>xxx</view>
</view>

子布局默认都是内联元素，即使设置display:block也不会自动扩充； 设置 flex: 1（即flex-grow=1）使能自动扩充；

# 纵向布局

<view class="flex flex-direction">
    <view>xxx</view>
    <view>xxx</view>
</view>

子布局默认是块级元素，回自动充满父布局；
